# Pitch (3 minutos)

## Roteiro Sugerido

### 1. O Problema (30 seg)

> Dor do cliente

Ao longo da minha trajetória em gestão financeira, percebi que o principal obstáculo das pessoas não é a ausência de ferramentas, mas a falta de clareza sobre a própria situação financeira. Dados indicam que 60% dos brasileiros desconhecem o conceito de reserva de emergência. O dinheiro circula, as obrigações chegam, e ao fim do mês, compreender para onde os recursos foram continua sendo um desafio real. O que as pessoas precisam não é de terminologia técnica complexa, mas de um apoio prático e acessível.

### 2. A Solução (1 min)

> Agente resolvendo o problema

Foi a partir dessa constatação que desenvolvi o **Tostão** — um assistente financeiro inteligente, impulsionado por IA Generativa, com foco em educação personalizada. Ele analisa o extrato do usuário, interpreta o fluxo de caixa e apresenta os resultados de forma clara, utilizando os próprios hábitos de consumo como referência. Tudo isso com responsabilidade: o Tostão não oferece recomendações de investimento diretas. Seu papel é o de um educador financeiro disponível a qualquer momento.

### 3. Demonstração (1 min)

> Funcionamento do agente

Na interface — desenvolvida em Python com Streamlit — o Tostão já opera com o contexto financeiro do usuário carregado. Ao consultar quanto foi gasto com alimentação, o sistema identifica automaticamente as transações relevantes, como supermercado e restaurantes, e apresenta uma resposta consolidada. Igualmente importante são os **guardrails** do sistema: caso o usuário solicite onde aplicar todo o seu capital, o Tostão redireciona a conversa para o seu papel educativo, explicando como os diferentes produtos financeiros funcionam, em vez de indicar uma opção específica.

### 4. Diferencial e Impacto (30 seg)

> Inovação da minha solução

Do ponto de vista técnico, o Tostão opera com arquitetura LLM executada integralmente de forma local. Isso elimina custos recorrentes com APIs de terceiros e, mais relevante ainda, garante que nenhum dado sensível — como extratos e perfil financeiro — seja transmitido para servidores externos. A privacidade do usuário é preservada em todas as etapas. O Tostão demonstra que, com Engenharia de Prompt consistente e uma arquitetura de dados bem estruturada, é possível democratizar a educação financeira de maneira segura e eficiente.
